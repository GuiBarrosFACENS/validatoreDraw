

-- 
validação

Vamos analisar cada um dos testes na classe WhitelistValidationTest:

testValidate_AllFields:

Este teste verifica se não há campos inválidos em uma fatura válida.
Executa o método validateFields da classe WhitelistValidation com uma fatura válida.
Verifica se o mapa de campos inválidos está vazio.
Caso contrário, lança uma mensagem de erro indicando que o campo não pode ser nulo ou vazio.
testValidate_UnpaidInvoice:

Verifica se há campos inválidos em uma fatura não paga.
Define o status de pagamento da fatura como não pago e o valor pago como zero.
Executa o método validateFields com a fatura não paga.
Verifica se há campos inválidos no mapa retornado.
Espera-se que o mapa não esteja vazio e que contenha a chave "payedValue".
testValidate_EmptyRequiredFields:

Testa se há campos inválidos quando campos obrigatórios estão vazios.
Define o valor pago como 1 e a mensagem como vazia.
Executa o método validateFields com a fatura modificada.
Verifica se há campos inválidos no mapa retornado.
Espera-se que o mapa não esteja vazio e que contenha a chave "message".
testValidateFields_BigDecimalLessThanOrEqualToZero:

Testa se há campos inválidos quando um campo BigDecimal é menor ou igual a zero.
Define o valor pago e o código do banco como zero ou negativo.
Executa o método validateFields com a fatura modificada.
Verifica se há campos inválidos no mapa retornado.
Espera-se que o mapa contenha as chaves "payedValue" e "bankCode".
testValidateFields_BigDecimalGreaterThanZero:

Testa se não há campos inválidos quando um campo BigDecimal é maior que zero.
Define o valor pago e o código do banco como positivos.
Executa o método validateFields com a fatura modificada.
Verifica se não há campos inválidos no mapa retornado.
testValidate_NullFields:

Testa se há campos inválidos quando um campo obrigatório é nulo.
Define a mensagem como nula.
Executa o método validateFields com a fatura modificada.
Verifica se há campos inválidos no mapa retornado.
Espera-se que o mapa contenha a chave "message".
testIsNullOrEmpty_BigDecimalZero:

Testa se um BigDecimal zero é considerado inválido.
Define o valor pago como zero.
Executa o método validateFields com a fatura modificada.
Verifica se o mapa de campos inválidos contém o campo "payedValue".
testIsNullOrEmpty_BigDecimalPositive:

Testa se um BigDecimal positivo é considerado válido.
Define o valor pago como positivo.
Executa o método validateFields com a fatura modificada.
Verifica se o mapa de campos inválidos não contém o campo "payedValue".
testIsNullOrEmpty_NonStringNonBigDecimal:

Testa se um campo não padrão (não String e não BigDecimal) não é considerado inválido.
Adiciona um campo não padrão à lista de campos.
Define o valor pago como positivo.
Executa o método validateFields com a fatura modificada.
Verifica se o mapa de campos inválidos não contém o campo não padrão.
